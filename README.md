## 1. Аннотация программного продукта

Данный репозиторий содержит исходный код классической логической игры «4 в ряд» (Connect Four) для двух игроков. Программный продукт разработан на платформе .NET 8 с использованием объектно-ориентированного языка программирования C# и технологии построения графического интерфейса Windows Forms.

Целью проекта является практическая реализация алгоритмов поиска паттернов на двумерных массивах, применение принципов SOLID для жесткого разделения бизнес-логики и пользовательского интерфейса (Layered Architecture), а также создание плавной анимации средствами GDI+.

## 2. Теоретические основы и математическая модель

В основе функционирования системы лежит алгоритм проверки победных комбинаций на игровом поле размером 6 строк на 7 столбцов. Дискретное пространство состояний описывается комбинаторикой и теорией игр с полной информацией.

Общее количество ячеек поля составляет:

`N = Rows × Columns = 6 × 7 = 42`

Максимальное количество ходов в одной партии равно 42 (при ничьей). Пространство состояний игры конечно и полностью определено. Каждая ячейка поля может находиться в одном из трёх состояний: пусто (0), занята Игроком 1 (1), занята Игроком 2 (2). Теоретическая верхняя граница числа состояний составляет 3⁴², однако реально достижимых позиций значительно меньше из-за ограничений механики «гравитации» фишек.

Ключевым алгоритмическим элементом является проверка победы после каждого хода. В отличие от наивного полного перебора (сложность O(Rows × Columns × 4)), в проекте реализован оптимизированный алгоритм, основанный на направленном обходе от последней сыгранной ячейки. Победная линия обязательно проходит через последнюю фишку, что позволяет проверить только четыре направления из этой точки:

- Горизонталь
- Вертикаль
- Диагональ \
- Диагональ /

Сложность данного подхода составляет O(WinLength × 4) = **O(1)** относительно размера поля.

В таблице 1 приведены параметры игрового поля и алгоритма.

**Таблица 1 — Параметры игровой модели**

| Параметр | Значение | Описание |
|----------|----------|----------|
| Размер поля | 6×7 | 6 строк, 7 столбцов |
| Длина победной линии (WinLength) | 4 | Количество фишек для победы |
| Количество направлений проверки | 4 | Горизонталь, вертикаль, две диагонали |
| Сложность алгоритма проверки победы | O(1) | Не зависит от размера поля |

## 3. Архитектурные решения и структуры данных

Проект спроектирован с соблюдением принципов SOLID и разделен на три независимые сборки:

1.  **ConnectFour.Logic** — слой предметной области. Содержит математический аппарат проверки победы, систему управления игровым полем (`GameBoard`), алгоритмы поиска линий (`WinChecker`) и игровой движок (`GameEngine`). Классы логики инкапсулированы и не имеют зависимостей от графической подсистемы.
2.  **ConnectFour** — слой представления (Windows Forms). Отвечает исключительно за рендеринг игрового поля, анимацию падения фишек, обработку пользовательского ввода (клики мыши, наведение) и диалоговые окна.
3.  **ConnectFour.Tests** — слой модульного тестирования. Содержит 31 автоматизированный тест на базе фреймворка xUnit, покрывающий критически важные сценарии бизнес-логики.

### Обоснование выбора типов данных

Для хранения состояния игрового поля выбран двумерный массив `int[,]` с фиксированными размерами. Значения ячеек кодируются целыми числами: 0 — пусто, 1 — фишка Игрока 1, 2 — фишка Игрока 2. Выбор обусловлен прямой адресацией за O(1) и отсутствием необходимости в динамическом изменении размера.

Для представления цвета фишки игрока использован тип перечисления `enum PlayerColor` (Red, Yellow). Это делает код самодокументируемым и защищённым от «магических чисел».

Для передачи результата хода между слоями логики и представления используется неизменяемый (immutable) класс `MoveResult` с инициализаторами `init`. Для хранения выигрышных ячеек применяется интерфейс `IReadOnlyList<(int Row, int Col)>`, обеспечивающий безопасную передачу коллекции в UI без риска её модификации.

### Устранение технических трудностей

В процессе разработки решены следующие проблемы:
- **Мерцание графики** — создан класс `DoubleBufferedPanel` с включенной двойной буферизацией (`OptimizedDoubleBuffer`), что полностью устранило визуальные артефакты при перерисовке.
- **Некорректный переход между формами** — реализована явная подписка на событие `FormClosed` для корректного возврата в главное меню.
- **Действия во время анимации** — введен флаг `_animating`, блокирующий обработку кликов и движений мыши до завершения падения фишки.

## 4. Модульное тестирование

Верификация и валидация алгоритмов проведена с использованием фреймворка `xUnit.net`. Набор тестов спроектирован по паттерну AAA (Arrange — Act — Assert). Все тесты изолированы от графического интерфейса и тестируют исключительно слой `ConnectFour.Logic`.

**Структура тестов (28 тестов):**
- `GameBoardTests` (8 тестов) — проверка механики «гравитации», заполнения столбцов и сброса поля.
- `WinCheckerTests` (6 тестов) — валидация обнаружения победных линий по всем четырём направлениям.
- `GameEngineTests` (11 тестов) — тестирование интеграции, смены игроков, обработки состояний (победа, ничья).
- `PlayerTests` (3 теста) — проверка создания игроков и валидации входных данных (пустое имя).

Произведено покрытие следующих сценариев:
- Проверка целостности алгоритма поиска выигрышных комбинаций (горизонталь, вертикаль, диагонали).
- Валидация граничных условий (попытка хода до старта игры, заполнение столбца до предела).
- Корректность смены текущего игрока после успешного хода.
- Обработка ничьей при полном заполнении поля.

## 5. Инструкция по сборке и запуску

Для компиляции и запуска программного продукта необходима среда разработки, поддерживающая платформу .NET 8 (рекомендуется Microsoft Visual Studio 2022 или JetBrains Rider).

**Порядок запуска:**

1.  Клонировать репозиторий в локальную файловую систему:
    ```bash
    git clone https://github.com/kinomatsu/ConnectFour.git
    ```
2. Открыть файл решения `.sln` в IDE.

3. Установить проект `ConnectFour` (Windows Forms приложение) в качестве загружаемого проекта по умолчанию.

4. Выполнить сборку решения (Build Solution, клавиша `F6`).

5. Запустить приложение в режиме отладки (Start Debugging, клавиша `F5`).

---

### Запуск модульных тестов:

- В Visual Studio: меню `Test → Run All Tests` (или комбинация `Ctrl+R`, `A`).

- В JetBrains Rider: `Run → Run All Tests from Solution`.

- Тестовые сборки располагаются в проекте `ConnectFour.Tests`.
